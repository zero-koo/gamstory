# gamstory 디자인 의뢰

## 1. 제품 한 줄 소개

**보드게임 플레이 기록 앱.** 계정 없이 바로 내 폰에 플레이를 기록하고, 로그인하면 친구들과 워크스페이스에서 기록을 공유·반응·댓글로 즐기는 로컬 우선(local-first) 웹앱.

## 2. 핵심 컨텍스트 (디자인 결정에 영향)

- **모바일 우선, 반응형.** 카카오톡·네이버 인앱 웹뷰와 Safari/Chrome에서 정상 렌더링되어야 함. 한 손 엄지 조작 동선이 최우선. 데스크톱은 넓은 레이아웃으로 우아하게 확장.
- **기본 언어 한국어(ko-KR), 영어(en) 동시 지원.** 한글 가독성이 1순위. 텍스트 길이가 언어별로 달라지므로 버튼·라벨이 늘어나도 깨지지 않는 레이아웃.
- **라이트/다크 모드 모두 지원.**
- **두 가지 사용 단계가 자연스럽게 이어져야 함:** ① 로그인 없이 혼자 쓰기 → ② 로그인 후 친구와 공유. "지금 로컬 기록 / 공유됨" 상태가 시각적으로 구분돼야 함.

## 3. 비주얼 무드 — 플레이풀 & 따뜻함

- 보드게임을 둘러앉아 즐기는 **친구들 사이의 따뜻하고 즐거운 감성.** 딱딱한 생산성 툴이 아니라, 추억을 남기는 가벼운 즐거움.
- 따뜻한 색온도의 컬러풀한 팔레트(예: 테라코타·머스터드·세이지·크림 톤 계열 — 자유롭게 제안), 부드러운 라운드 코너, 손맛 있는 일러스트/이모지 활용.
- 단, **귀엽기만 하고 정보가 안 보이는 함정은 피할 것.** 기록 데이터(승자, 점수, 날짜, 참가자)가 명확히 읽혀야 함. 플레이풀함은 "톤"이고, 가독성은 "원칙".
- 빈 화면(empty state)을 즐겁게 — 첫 기록을 유도하는 일러스트와 카피.

## 4. 디자인할 화면 (v1 전체)

### 익명/로컬 단계

1. **목록 뷰 (홈)** — 플레이 카드 리스트. 카드마다 게임명·날짜·참가자·승자·사진 썸네일·동기화 상태 배지. 게임/멤버/기간 필터 + 정렬.
2. **캘린더 뷰** — 월 그리드. 날짜 칸에 플레이 개수 배지 + 사진 썸네일 2장 미리보기. 날짜 탭하면 그날 기록이 바텀 드로어로.
3. **플레이 작성/수정 폼** — 게임 선택(검색+새로 만들기), 멤버 다중 선택(칩+인라인 생성), 날짜, 참가자별 순위·점수·승자 토글, 메모, 사진 첨부(최대 5장).
4. **플레이 상세** — 사진 갤러리, 참가자 결과표, 메모. (공유된 플레이면) 리액션 바 + 댓글 스레드.

### 로그인/워크스페이스 단계

5. **로그인** — Google·Kakao 버튼. 인앱 웹뷰 대응용 "브라우저에서 열기" 안내.
6. **워크스페이스 목록** — 내가 속한 그룹들.
7. **워크스페이스 상세** — 그 그룹의 공유된 플레이 피드(목록/캘린더 재사용).
8. **워크스페이스 설정** — 멤버 관리, 초대 링크 생성/복사/만료/취소.
9. **초대 수락** — 링크로 진입 시 "○○ 워크스페이스에 참여" 화면.

### 공통 컴포넌트

- 리액션 바: 고정 이모지 5종 (👍 ❤️ 🎉 😂 👏), 토글, 개수 표시.
- 댓글: 플랫(스레드 없음), 작성/수정/삭제, 삭제된 댓글 tombstone.
- 동기화 상태 배지: `로컬에만 있음` / `업로드 중` / `공유됨` / `실패(재시도)`.
- 사진 업로더: 진행률, 5MB·5장 제한 안내, 에러 인라인 배너.
- 토스트 / 인라인 배너 / 전체화면 복구 상태 3종 피드백 패턴.
- 워크스페이스 전환기(스위처), 언어 전환, 라이트/다크 토글.

## 5. 디자인 시스템 산출물

- **컬러 토큰** (라이트/다크): background, foreground, primary, muted, border + 시맨틱(success/warning/danger) + 리액션 강조색. 현재 코드가 CSS 변수 기반이므로 토큰 네이밍으로 정의.
- **타이포 스케일** — 한글·영문 모두 검증. 시스템 폰트 우선(웹뷰 호환), 필요 시 본문 폰트 제안.
- **간격·라운드·그림자·아이콘 스타일** 규칙.
- **핵심 컴포넌트 세트**: Button, Card, Input, Select, Dialog, Drawer, Form, Chip, Badge, Avatar, Calendar cell, Reaction button, Comment item, Empty state.
- 각 컴포넌트의 상태(기본/hover/focus/disabled/loading/error)와 모바일·데스크톱 반응형.

## 6. 기술 제약 (구현 연계)

- **Tailwind v4 + shadcn/ui** 기반. 토큰은 CSS 변수로 매핑 가능해야 함.
- 컬러/타이포 값은 코드에 바로 옮길 수 있게 **구체 수치**로 제시(hex, rem, px).
- 아이콘은 **lucide-react** 우선, 부족하면 이모지/커스텀 일러스트.
- 외부 폰트/이미지 CDN 의존 최소화(웹뷰·CSP 환경).

## 7. 받고 싶은 것 (Deliverables)

1. 무드보드 + 컬러/타이포 토큰 정의(라이트·다크).
2. 위 9개 화면의 **모바일 + 데스크톱** 시안(주요 상태 포함: 데이터 있음/빈 화면/에러).
3. 핵심 컴포넌트 시트.
4. 한국어 기준으로 디자인하되 영어 전환 시 깨지지 않음을 1개 화면으로 시연.
